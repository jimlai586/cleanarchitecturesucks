Less is more

You know what’s the first thing I consider when I’m designing a library?

How to make it easy to use.

I don’t want boilerplate setup or extra object life cycles to maintain. I want it to be self-explanatory or lightweight enough you can easily figure things out.

I want you to write as few codes as possible, with minimum learning curve. I want any fresh-graduate to be able to jump in and start contributing as soon as possible. I want almost 0 efforts spent on managing dependency. I want most efforts spent on getting the job done, not getting the boilerplate done.

Your goal should be to create a code base that is both easy to maintain and productive. In order to do that you need to have a shrewd understanding of language features and exploit that; and you need to plan your strategy according to project objectives and edge cases.

You won’t find any of these considerations in those MVVM/RxSwift/VIPER guides.

But you will find entertainment.
